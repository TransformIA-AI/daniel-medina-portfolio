## 4. Demo brutal: AI Interview Concierge + HITL Cockpit

### Concepto

La web debe incluir un agente llamado:

```text
Daniel AI Interview Concierge
```

No debe fingir ser Dani. Debe decir:

> “Soy un asistente público de portfolio. Puedo ayudarte a entender el encaje profesional de Daniel, recomendarte el CV adecuado, preparar una entrevista y enviar una solicitud para revisión humana.”

### Funciones del agente

#### Fase 1: demo segura, sin secretos

- Responde preguntas sobre el perfil usando solo datos públicos del portfolio.
- Recomienda CV según rol:
  - AI Transformation
  - Operations / Business Manager
  - Digital Workplace / Public Sector
  - AI Enablement / HITL
- Genera un resumen de encaje para una oferta pegada por el recruiter.
- Crea un “Interview Request Draft”.
- Muestra un panel de “Pending Dani approval”.
- Genera un Evidence Receipt:
  - pregunta recibida
  - datos usados
  - respuesta generada
  - riesgos
  - siguiente acción
  - estado: draft only / human approval required

#### Fase 2: n8n + formularios

- Formulario de recruiter.
- Webhook n8n.
- Registro en Google Sheets/Notion/Airtable.
- Email draft para Dani.
- Notificación Telegram/WhatsApp/Email.
- No enviar emails automáticos sin aprobación.

#### Fase 3: calendario

Dos opciones:

1. **Modo seguro:** enlace a Calendly/Google Calendar appointment schedule.
2. **Modo avanzado HITL:** Google Calendar FreeBusy para proponer slots, pero crear evento solo tras aprobación explícita de Dani.

### Guardrails

El agente debe rechazar:

- Inventar experiencia.
- Compartir teléfono/correo si no está en página pública.
- Prometer disponibilidad real si no ha consultado calendario.
- Aceptar entrevistas sin aprobación.
- Exponer secretos, tokens, repos privados o material cliente.
- Presentarse como Daniel humano.
